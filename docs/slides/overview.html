見ながら理解する。
焼き込まず、重ねるだけ。

  • 任意サイトで再生中の多言語動画を、見ながら日本語で理解するのが目的。 動画はブラウザ内でローカル再生されたまま、どこにも送らない。
  • 字幕の焼き込み・動画ファイル生成はしない（翻案物の生成を避ける）。再生時オーバーレイのみ・保存しない。 著作権上のスタンス。和訳は透明レイヤーで一時表示するだけ。
  • 体験は「数秒遅れて流れるライブ字幕」。暫定（薄字）→ 確定（濃字）の2段で出す。 全画面でも見えるよう、字幕は全画面要素の中へ動的に差し込む。

02 — アーキテクチャ
03 / 06

重い処理は Spark、Mac は「拾う・重ねる」だけ

the Mac client Chrome 拡張

  • service worker / offscreen … tabCapture で音声取得 → Opus 圧縮 → WSS 送信
  • content script … 透明オーバーレイ描画に専念（サーバを直接叩かない）
  • ネットワークは SW/offscreen に集約 → mixed content / PNA を回避

the Spark 推論ゲートウェイ（CUDA）

  • FastAPI + WSS … 音声を受け、和訳テキストを返す
  • faster-whisper / Whisper large-v3 … 原語 STT
  • Qwen 系 LLM・Swallow 70B（vLLM / Ollama 常駐）… 和訳
上り
音声のみ Opus 16–24 kbps
下り
和訳テキストのみ
経路は Tailscale（mesh）。自宅 Wi-Fi でも外出テザリングでも同一エンドポイントで到達。
03 — 推論パイプライン
04 / 06

STT → 翻訳の2段構成

[1] 音声認識（STT）

  • Whisper large-v3 を CUDA で実行（transformers / PyTorch）
  • Silero VAD で発話単位に分割してから 1 発話ずつ転写
  • Whisper の translate は英訳専用 → 和訳には使えない

[2] 翻訳（LLM）

  • Qwen 系 Instruct / Swallow 70B を vLLM・Ollama で常駐
  • 専用翻訳モデルより日本語の流暢さで有利
  • 直近の (原文, 訳文) を文脈として渡し、訳の辻褄をそろえる
  • 長い無音でターン交代マーカーを付与（簡易な話者切替の目印）
128GB ユニファイドメモリ（DGX Spark / Blackwell）で「ローカルなのに高品質」が成立。
04 — 設計上の要（ガード）
05 / 06

崩してはいけない 4 つの軸

  • 役割分担を崩さない … Mac = 音声取得 + 描画 / Spark = 推論。重い処理は必ず Spark 側。
  • エンドポイントは Tailscale MagicDNS 名で一本化 … 自宅/外出で切替えない・IP 直打ちしない。tailscale serve で正規 HTTPS/WSS 終端。
  • ネットワーク処理は SW / offscreen に集約 … content script は描画専任。mixed content / Private Network Access を回避。
  • 秘匿値はコミットしない … 実機名・IP・tailnet 名・鍵は .env / CLAUDE.local.md。コードでは *.ts.net プレースホルダ。
公開方針: 安定・汎用化後に Private → Public（履歴も引き継がれる前提で同基準を徹底）。
